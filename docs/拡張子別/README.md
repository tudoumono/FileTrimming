# 拡張子別メモ README

作成日: 260311 214114
更新日: 260311 214114

## 1. 位置づけ

- 現行実装の正本は [ドキュメント処理パイプライン 詳細設計書 v2](../ドキュメント処理パイプライン詳細設計書_v2.md) とする
- `docs/拡張子別/*.md` は、拡張子ごとの補足論点、品質リスク、試験観点、将来拡張候補を整理するためのメモとする
- 記述に差分が出た場合は、現行挙動については詳細設計書 v2 を優先する

## 2. 記述ルール

- 現行実装の処理ルートを書く場合は要約に留め、Phase 単位の仕様やログ項目は詳細設計書 v2 を参照する
- 未実装案や代替案は `将来拡張候補` `未実装` `要検証` のいずれかで明示する
- 形式固有の論点、サンプル依存の注意、試験時の確認項目は拡張子別メモに残してよい

## 3. 読み方

- まず詳細設計書 v2 で現行フローを確認し、その後に対象拡張子のメモを読む
- `.doc` `.xls` のようなレガシー形式は、正規化後に基準形式へ合流する前提で簡潔に保つ
- 実装変更時は、まず詳細設計書 v2 を更新し、必要な場合のみ該当メモを追随させる
